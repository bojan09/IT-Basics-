📖 EXPLAIN LIKE I'M A BEGINNER

What is PowerShell?

💡 THE ANALOGYThink of PowerShell as a remote control for your computer — but instead of pressing buttons, you type commands. Unlike a TV remote that sends simple signals, PowerShell sends full structured instructions and gets back structured data you can further process. It's automation on steroids.
⚡ KEY INSIGHTPowerShell works with objects, not text. When you run Get-Process, you get a list of process objects with properties (Name, CPU, Memory). You can then filter, sort, and manipulate them — not just read text output.
⌨️ ESSENTIAL CMDLETS
POWERSHELL
# Get help for any command
Get-Help Get-Process -Full

# List running processes
Get-Process
Get-Process | Where-Object CPU -gt 50

# File system
Get-ChildItem # like ls / dir
Get-Content C:\log.txt # like cat
Set-Content file.txt "Hello" # write to file
Copy-Item src.txt dest\
Remove-Item oldfile.txt

# Services
Get-Service
Start-Service Spooler
Stop-Service Spooler

# Network
Test-NetConnection google.com -Port 443
Get-NetIPAddress
Resolve-DnsName google.com
📜 THE PIPELINE — POWERSHELL'S SUPERPOWER

Chain commands together. Each command receives the object output of the previous one.

POWERSHELL — PIPELINES
# Find the top 5 CPU-hungry processes
Get-Process | Sort-Object CPU -Descending | Select-Object -First 5

# Find all stopped services and export to CSV
Get-Service | Where-Object Status -eq 'Stopped' | Export-Csv stopped.csv

# Find large files over 100MB
Get-ChildItem C:\ -Recurse | Where-Object Length -gt 100MB | Sort-Object Length
📋 YOUR FIRST SCRIPT
POWERSHELL — SCRIPT
# system-report.ps1
# Generates a simple system health report

$report = @{}

# OS info
$os = Get-ComputerInfo
$report.OS = $os.OsName
$report.Uptime = (Get-Date) - $os.OsLastBootUpTime

# Disk space
$disk = Get-PSDrive C
$report.FreeGB = [math]::Round($disk.Free / 1GB, 2)

# Output
$report | Format-Table
Write-Host "Report complete!" -ForegroundColor Green
⚠️ COMMON MISTAKES

What beginners get wrong

❌ MISTAKE 1Confusing PowerShell with CMD. They look similar but are completely different. PowerShell uses objects; CMD uses text strings. dir works in both but returns very different data.
❌ MISTAKE 2Not using -WhatIf before destructive commands. Add -WhatIf to see what would happen without actually doing it. Essential for Remove-Item, Stop-Service, etc.
❌ MISTAKE 3Ignoring execution policy. Scripts won't run by default. Set: Set-ExecutionPolicy RemoteSigned — but understand what it means first.
❌ MISTAKE 4Not using Get-Help. Every cmdlet has built-in documentation. Get-Help Get-Process -Examples shows you real usage examples.
💡 Quick Summary — PowerShell
PowerShell works with objects, not plain text — this makes filtering and manipulation much more powerful than CMD.
Cmdlets follow a Verb-Noun pattern: Get-Process, Stop-Service, New-Item, Remove-Item.
The pipeline | chains cmdlets together — each receives the object output of the previous.
Use Where-Object to filter and Select-Object to pick specific properties.
Always use -WhatIf to preview destructive actions before running them for real.
