💡 THE ANALOGYUnix is like LEGO. Each Unix command is a small, simple brick that does one thing well. The magic is in combining those bricks — connecting commands together to build complex structures. This philosophy of small, composable tools is still the backbone of modern computing 50+ years later.
🧩 THE UNIX PHILOSOPHY — 3 RULES
🎯
Do one thing well
ls lists files. grep searches text. sort sorts. Each tool has exactly one job and does it excellently.
🔗
Work together
Commands are designed to be chained via pipes. The output of one is the input of another. Build complexity from simplicity.
📄
Text is universal
Everything is a file or a text stream. Config files are plain text. Logs are plain text. This universality enables composability.

🔧 CORE UNIX TOOLS — STILL USED DAILY
BASH / SH
# Text processing — Unix's greatest strength
grep "ERROR" /var/log/syslog # search text
sed 's/old/new/g' file.txt # find/replace
awk '{print $1, $3}' data.txt # extract columns
cut -d',' -f1 data.csv # cut CSV column 1
sort names.txt | uniq # sort and deduplicate
wc -l file.txt # count lines

# The classic Unix pipeline — count ERROR lines
cat /var/log/syslog | grep "ERROR" | wc -l

# Find + process in one line
find /logs -name "*.log" -mtime +30 | xargs rm

⚙️ PROCESSES AND SIGNALS
BASH
# View processes
ps aux | grep nginx
pgrep nginx # just get PIDs

# Send signals
kill -15 <PID> # SIGTERM — graceful stop
kill -9 <PID> # SIGKILL — force stop
kill -1 <PID> # SIGHUP — reload config

# Background jobs
sleep 60 & # run in background
jobs # list background jobs
fg %1 # bring job to foreground
⚠️ COMMON MISTAKES

What beginners get wrong

❌ MISTAKE 1Thinking Linux = Unix. Linux is Unix-like and follows Unix philosophy, but it's not Unix. macOS is actually certified Unix. Linux is POSIX-compliant but a separate family.
❌ MISTAKE 2Using SIGKILL (-9) for everything. Always try SIGTERM (-15) first — it lets the process clean up. SIGKILL (-9) doesn't let the process do anything before dying.
❌ MISTAKE 3Writing non-portable shell scripts. Bash features don't work in sh. Use #!/bin/bash explicitly or stick to POSIX sh if you need portability.
❌ MISTAKE 4Underestimating text tools. grep, awk, sed, sort, cut, uniq — beginners skip these. Mastering them makes you dramatically more productive than any GUI tool.
💡 Quick Summary — Unix Fundamentals
Unix philosophy: small tools that do one thing well, designed to work together via pipes.
Everything is a file — devices, processes, network sockets. This universality powers Unix's composability.
The pipe | connects commands — the output of one becomes input to the next.
Master grep, awk, sed, sort, cut, wc — these text tools are the Swiss Army knife of Unix.
Linux and macOS are both Unix descendants — Unix fundamentals apply directly to both.
