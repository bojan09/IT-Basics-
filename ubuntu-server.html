🖥️ Ubuntu Server Basics

Set up, secure, and manage an Ubuntu Server — SSH, firewall, services, and essential admin tasks.

📖 EXPLAIN LIKE I'M A BEGINNER

What is Ubuntu Server?

💡 THE ANALOGYUbuntu Server is like renting an empty office building. You get the space (hardware), the plumbing (network), and electricity (power) — but you bring everything else. No GUI. Just a terminal. You're the architect.
🚀 FIRST STEPS AFTER INSTALL
1
Update the system
Always update first — security patches and package upgrades.
2
Create a non-root user
Never run a server as root day-to-day. Create a user and add to sudo group.
3
Configure SSH key auth
Disable password login. Use SSH keys — dramatically more secure.
4
Enable the firewall
UFW (Uncomplicated Firewall) — allow only the ports you need.
5
Install fail2ban
Auto-bans IPs that fail too many login attempts. Essential for public servers.
BASH — INITIAL SERVER SETUP
# 1. Update everything
sudo apt update && sudo apt upgrade -y

# 2. Create a new user
sudo adduser deploy
sudo usermod -aG sudo deploy

# 3. Configure UFW firewall
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status

# 4. Install fail2ban
sudo apt install fail2ban -y
sudo systemctl enable fail2ban
🔑 SSH KEY AUTHENTICATION
BASH
# On your LOCAL machine — generate a key pair
ssh-keygen -t ed25519 -C "your@email.com"

# Copy public key to server
ssh-copy-id deploy@your-server-ip

# On SERVER — disable password auth
sudo nano /etc/ssh/sshd_config
# Set: PasswordAuthentication no
sudo systemctl restart ssh
⚙️ SYSTEMD SERVICE MANAGEMENT
BASH
sudo systemctl status nginx # check status
sudo systemctl start nginx # start service
sudo systemctl stop nginx # stop service
sudo systemctl enable nginx # start on boot
sudo systemctl restart nginx # restart
sudo journalctl -u nginx -f # follow logs
⚠️ COMMON MISTAKES

What beginners get wrong

❌ MISTAKE 1Disabling password auth before testing SSH keys work. Always verify key login in a second terminal before closing your first session.
❌ MISTAKE 2Not setting up automatic security updates. Install unattended-upgrades for automatic security patches.
❌ MISTAKE 3Leaving default ports open with no firewall. Lock down everything. Only open ports your services actually need.
❌ MISTAKE 4Not monitoring logs. Check journalctl and /var/log/auth.log regularly — failed logins are a warning sign.
💡 Quick Summary — Ubuntu Server
Always update first, create a non-root user, and set up SSH key authentication.
Use UFW to manage firewall rules — default deny, allow only what's needed.
systemd / systemctl manages all services — start, stop, enable, check status.
fail2ban protects against brute-force attacks automatically.
Monitor logs with journalctl -u <service> -f for real-time output.
